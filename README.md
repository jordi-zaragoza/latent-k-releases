# latent-k

Auto-sync context for AI coding assistants (Claude Code, Gemini CLI).

## Features

- Automatic context injection at session start
- Auto-sync on session end
- Multi-CLI support (Claude Code, Gemini CLI)
- Compressed LK format (~2-3k tokens)
- AI-powered file analysis

## Installation

```bash
curl -fsSL https://github.com/jordi-zaragoza/latent-k-releases/releases/download/v1.0.6/install.sh | bash
```

Binary is installed to `/usr/local/bin/lk`.

## Quick Start

```bash
lk activate  # Enter license key
lk setup     # Configure AI provider
lk enable    # Enable hooks for Claude + Gemini
lk sync      # Initial sync
```

## How it works

```
.lk/
├── syntax.lk      # LK grammar (encrypted)
├── project.lk     # Project metadata (encrypted)
├── ignore         # Project-specific ignore patterns
└── domains/
    ├── cli.lk     # CLI domain files (encrypted)
    └── core.lk    # Core domain files (encrypted)
```

## Commands

```bash
lk activate       # Activate license
lk setup          # Configure AI provider
lk sync           # Manual sync
lk status         # Show status
lk enable         # Enable hooks (both CLIs by default)
lk enable -t claude   # Enable for Claude Code only
lk enable -t gemini   # Enable for Gemini CLI only
lk disable        # Disable hooks
lk update         # Update to latest version
lk clean          # Remove lk data
```

### Ignore Patterns

**Global patterns** (in config):
- `**/node_modules/**`, `**/dist/**`, `**/.git/**`, `**/*.lock`

**Project patterns** (`.lk/ignore`):
- Auto-generated by AI on first sync
- Only project-specific patterns not covered by global
